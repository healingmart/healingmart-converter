[PDF → XLSX]
- PDF.js 텍스트 위치를 기준으로 같은 Y 좌표의 글자를 행으로 묶음
- 글자 사이 X 간격이 큰 지점을 별도 셀로 추정
- PDF 페이지 1개 = XLSX 워크시트 1개
- 최대 300페이지 / 전체 약 50,000행
- 원본 PDF의 실제 병합셀·수식·테두리를 복원하는 기능은 아님

[XLSX → PDF]
- SheetJS 0.18.5로 워크북 읽기
- 최대 20개 시트
- 전체 5,000행
- 9열 × 19행 단위로 A4 가로 PDF 페이지 분할
- 한글 안정성을 위해 Canvas 렌더링 후 PDF에 삽입
- 차트·이미지·피벗·조건부서식의 완전 재현은 하지 않음

[PDF → PPTX]
- PDF.js로 페이지를 렌더링
- PptxGenJS 4.0.1로 16:9 PowerPoint 생성
- PDF 페이지 1개 = 슬라이드 1개
- 페이지 이미지를 슬라이드 안에 비율 유지 배치
- 결과 객체는 개별 편집 가능한 글자/도형이 아니라 페이지 이미지
- 최대 100페이지

[PPTX → PDF / JPG / PNG]
- pptx-svg 0.6.4 PptxRenderer 사용
- PPTX를 브라우저에서 SVG로 정적 렌더링
- SVG를 Canvas로 변환한 뒤 PDF/JPG/PNG 생성
- 숨김 슬라이드 제외/포함 선택
- JPG/PNG 여러 장은 ZIP 다운로드
- 최대 100슬라이드 / PPTX 40MB
- 애니메이션·전환은 정적 출력에 포함되지 않음
