[DOCX → HWPX]
- Mammoth 1.12.0 브라우저 빌드로 DOCX → HTML 내용 구조 추출
- script/style/form/iframe 등 제거

- @ssabrojs/hwpxjs 0.4.0의 htmlToHwpx()로 표준 HWPX 생성
- DOCX 최대 30MB
- Word의 정밀한 페이지 배치·머리글/바닥글·도형은 단순화 가능

[PDF → HWPX]
- PDF.js 3.11.174 텍스트 레이어 사용
- 글자 Y/X 좌표로 줄 순서 정리
- 페이지별 제목 + 문단 HTML 생성
- @ssabrojs/hwpxjs htmlToHwpx()로 HWPX 생성
- PDF 최대 60MB / 300페이지 / 본문 100만 자
- 스캔 PDF는 OCR이 없으면 변환 불가
- PDF 화면 레이아웃을 HWPX로 그대로 복원하는 방식은 아님

[HWPX Writer]
- @ssabrojs/hwpxjs 0.4.0
- 공식 브라우저 ESM 번들:
  dist/browser/hwpxjs.browser.mjs
- 문서상 HWPX writer는 첫 STORED mimetype,
  META-INF/container.xml / manifest.xml,
  OPF manifest+spine,
  header.xml 등을 포함하는 OWPML 패키지를 생성

[TTF → OTF]
- opentype.js 2.0.0
- TTF를 parse() 후 Font.toArrayBuffer()로 OpenType 출력
- 출력 4바이트가 OTTO인지 확인
- OTTO가 아니면 .otf 다운로드를 중단해 잘못된 확장자 변환 방지
- 가변/컬러 폰트·고급 OpenType 레이아웃·힌팅 등 일부 특수 테이블은 손실 가능
