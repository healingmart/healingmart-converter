[HWP 브라우저 엔진]
- @rhwp/core 0.7.19
- Rust + WebAssembly 기반 HWP/HWPX 파서·렌더러
- HWP 5.0 바이너리 파일을 HwpDocument로 로드
- pageCount()로 페이지 수 확인
- renderPageSvg(pageIndex)로 페이지별 SVG 생성
- 첫 실행 시 rhwp_bg.wasm 로드
- 파일은 외부 변환 서버로 업로드하지 않음

[HWP → TXT]
- 페이지별 SVG에서 text 요소의 렌더링 텍스트 추출
- 페이지 구분 추가
- 표·도형·수식의 시각적 위치 정보는 TXT에 포함되지 않음

[HWP → HTML]
- 각 페이지의 renderPageSvg 결과를 HTML 문서에 그대로 배치
- 페이지형 HTML로 저장
- 정적인 시각 보존을 우선

[HWP → PDF]
- HWP 페이지 SVG → Canvas → jsPDF
- 한 페이지씩 PDF 페이지로 구성
- 최대 200페이지
- 설치 글꼴/브라우저 글꼴 접근 차이로 원본과 줄바꿈 차이 가능

[HWP → DOCX]
- 렌더링된 SVG 페이지의 텍스트를 추출
- 기본 DOCX 문단으로 구성
- 원본 HWP의 표·도형·페이지 레이아웃이 Word 편집 객체로 복원되는 방식은 아님

[AVIF 출력]

- JPG → AVIF
- PNG → AVIF

- 최대 2,500만 픽셀
- 고해상도 이미지는 AVIF 인코딩 시간이 길 수 있음
