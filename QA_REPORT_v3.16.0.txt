[HWPX 처리 방식]
- HWPX가 ZIP 기반 XML 문서인지 확인
- 루트 mimetype이 존재하면 application/hwp+zip 확인
- Contents/content.hpf의 manifest + spine 순서를 우선 사용
- content.hpf를 읽지 못하는 경우 Contents/section0.xml, section1.xml ... 순서로 폴백
- section XML의 문단 텍스트와 표 셀 텍스트 추출
- section XML을 찾지 못하거나 본문 블록이 없으면 Preview/PrvText.txt를 TXT 계열 폴백으로 사용

[HWPX → TXT]
- 문단은 빈 줄로 구분
- 표는 행 단위 줄바꿈 + 셀 탭 구분

[HWPX → HTML]
- 문단과 표를 기본 HTML로 변환
- 세밀한 한/글 글자 모양·페이지 배치·도형 위치는 재현하지 않음

[HWPX → DOCX]
- 최소 OOXML DOCX 패키지 생성
- 문단과 표 중심
- 기본 글꼴 맑은 고딕 지정
- 복잡한 원본 서식은 동일하게 유지되지 않을 수 있음

[HWPX → PDF]
- 본문과 표를 텍스트로 정리한 후 A4 PDF 생성
- 한글 안정성을 위해 Canvas 렌더링
- 결과 PDF 본문은 이미지 기반
- 원본 한/글 페이지 렌더링을 복제하는 기능이 아님

[안전 한도]
- HWPX 최대 50MB
- 내부 파일 최대 2,000개
- 구역 최대 100개
- 구역 XML 1개 최대 15MB
- 본문 최대 100만 자
- PDF 변환 최대 70만 자 / 300페이지
